## 1. 전체 아키텍처

```text
입력층
  Markdown 노트
  교재/슬라이드 PDF
  PPTX 슬라이드
  선택: Semantic Scholar / PubMed 논문 검색
        |
        v
처리층
  scripts/synthesize.py
  - 소스 수집
  - PDF/슬라이드 텍스트 추출
  - 분류/공백 감지
  - 교차 과목 분석
  - 선택형 논문 보강
  - LLM 합성
  - 퀴즈 생성
  - Markdown 저장
        |
        v
저장층
  output/md/
  queue/
  approved/
  rejected/
  weak_concepts.json
  cache/learning_history.json
  cache/hermes/
        |
        v
사용 표면
  review.py
  hermes.py
  dashboard/app.py
```

핵심 원칙:

| 원칙 | 내용 |
|---|---|
| 경로 중앙화 | `path_utils.get_study_paths()`를 통해 주요 경로를 계산 |
| LLM 단일 관문 | LLM 호출은 `LLMRouter.generate()` 경유 |
| 퀴즈 파일 일원화 | 퀴즈 저장/승인/거절은 `quiz_store.py` 경유 |
| 학습 이력 로컬화 | 학습 결과는 로컬 JSON에 저장 |
| 출력 단순화 | 정리본 출력은 Markdown만 사용 |

## 3. 파이프라인 흐름

진입점: `synthesize.process_note(note_path, config, logger)`

```text
process_note()
├─ detect_subject()
├─ collect_sources_for_note()
│  ├─ note_text
│  ├─ textbook_text
│  ├─ slides_text
│  ├─ textbook_images
│  └─ slides_images
├─ run_agents()
│  ├─ ClassifierAgent.classify()
│  └─ GapDetector.detect_gaps()
├─ run_cross_subject_analysis()
├─ run_paper_enrichment()
│  └─ papers.enabled=false 이면 즉시 skip
├─ build_synthesis()
│  ├─ synthesize_notes()
│  └─ pubmed.enabled=true 이면 PubMed 섹션 추가
├─ append_analysis_sections()
├─ append_study_plan_section()
├─ append_quiz_sections()
│  ├─ textbook_quiz.add_textbook_quiz_section()
│  └─ generate.process_content()
└─ persist_outputs()
   ├─ save_synthesis_md()
   ├─ record_synthesis_session()
   └─ refresh_hermes_schedule()
```

PDF 출력 단계는 제거되었다. `save_synthesis_pdf()`는 오래된 호출과 테스트 호환을 위해 no-op으로 남아 있으며 새 출력물을 만들지 않는다.

## 4. LLM 라우팅

`LLMRouter`가 사용하는 provider:

| Provider | 용도 |
|---|---|
| LM Studio | 로컬 초안, 분류, 추출, 반복 작업 |
| ChatGPT | 분석, 보충 설명, 교차 과목 연결 |
| Claude | 심화 합성, 퀴즈 생성, 고난도 설명 |

외부 router provider는 제거되었다. `config.yaml`에는 `router_tasks`나 외부 router endpoint를 두지 않는다.

JSON 응답이 필요한 에이전트는 `LLMRouter.generate_json()`을 사용한다. 내부적으로 `generate()` 결과에서 JSON block 또는 중괄호 영역을 추출하고 `json.loads()`를 시도한다.

## 5. 논문 보강

논문 보강은 선택 기능이다.

| 설정 | 처리 |
|---|---|
| `papers.enabled: false` | Semantic Scholar 보강 skip |
| `papers.enabled: true` | `paper_fetcher.fetch_papers_for_note()` 실행 |
| `pubmed.enabled: false` | PubMed 섹션 skip |
| `pubmed.enabled: true` | `pubmed_client.search_and_summarize()` 실행 |

기본값은 OFF다. GUI의 Feature 탭에서도 켜고 끌 수 있다.

## 6. 메모리와 복습 로직

메모리는 로컬 JSON만 사용한다.

```text
MemoryManager
├─ weak_concepts.json
│  └─ 과목별 concept mastery와 복습 예정일
└─ cache/learning_history.json
   └─ 최근 학습 이벤트
```

`ConceptEntry` 주요 필드:

| 필드 | 의미 |
|---|---|
| `encounter_count` | 개념을 만난 횟수 |
| `correct_count` | 정답 누적 |
| `last_encounter` | 마지막 학습 시각 |
| `mastery` | 숙련도 |
| `priority` | high / medium / low |
| `recent_mistakes` | 최근 오답 메모 |
| `sr_interval` | 간격반복 일수 |
| `sr_ease_factor` | SM-2 ease factor |
| `sr_next_review` | 다음 복습일 |

SM-2 요약:

| 결과 | interval | ease factor |
|---|---|---|
| correct | 첫 정답 6일, 이후 `interval * ef` | 증가 |
| partial | `interval * 0.7` | 소폭 감소 |
| wrong | 1일 | 감소 |

`get_due_reviews()`는 `sr_next_review <= today`인 개념을 우선순위와 숙련도 기준으로 정렬한다.

## 7. 퀴즈 저장 흐름

퀴즈는 `queue`, `approved`, `rejected` 폴더에서 관리한다.

```text
generate.process_content()
  -> queue/{quiz_id}.json
  -> queue/{quiz_id}.md

approve_quiz()
  -> approved/{quiz_id}.json
  -> approved/{quiz_id}.md
  -> queue 원본 삭제

reject_quiz()
  -> rejected/{quiz_id}.json
  -> rejected/{quiz_id}.md
  -> queue 원본 삭제
```

퀴즈 복습 결과는 `MemoryManager.record_result()`로 전달되어 `weak_concepts.json`과 `cache/learning_history.json`에 반영된다.

## 9. 경로 해석 규칙

`path_utils.get_study_paths(config)`가 `StudyPaths` dataclass를 반환한다.

```python
vault = resolve_path(config["vault_path"], cwd)
notes_base = resolve_path(config["notes_dir"], vault)
pipeline = resolve_path(config["pipeline_dir"], vault)
```

상대 경로는 기준 경로에 붙여 해석하고, 절대 경로는 그대로 사용한다.

| 경로 | 의미 |
|---|---|
| `vault` | Obsidian vault |
| `notes_base` | 기본 노트 폴더 |
| `pipeline` | pipeline runtime root |
| `queue` | 검토 대기 퀴즈 |
| `approved` | 승인 퀴즈 |
| `rejected` | 거절 퀴즈 |
| `output_md` | Markdown 정리본 |
| `cache` | 논문/텍스트/Hermes/학습 이력 캐시 |
| `logs` | 실행 로그 |
