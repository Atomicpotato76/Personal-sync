## 1. 전체 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│  입력층                                                          │
│  Obsidian Vault (.md 필기) + 교재 PDF + 강의자료 PDF            │
└────────────────────┬────────────────────────────────────────────┘
                     │ watcher.py (파일 이벤트 감지)
                     ▼
┌─────────────────────────────────────────────────────────────────┐
│  처리층 — synthesize.py (10단계)                                │
│                                                                  │
│  소스수집 → PDF파싱 → 에이전트분류 → 교차과목 → 논문수집        │
│       → LLM종합 → 심화분석 → 학습계획 → 퀴즈생성 → 저장        │
│                                                                  │
│  LLM 라우터: LM Studio(로컬) → ChatGPT → Claude (3-tier)       │
└────────────────────┬────────────────────────────────────────────┘
                     │
          ┌──────────┼───────────┐
          ▼          ▼           ▼
    output/md/   queue/      weak_concepts.json
    output/pdf/  (퀴즈)      learning_history.json
    vault/정리/               cache/mem0/ (벡터 DB)
                     │
          ┌──────────┼───────────┐
          ▼          ▼           ▼
      review.py  dashboard/  mcp_server.py
      (CLI)      app.py      (Claude Desktop)
                 (Streamlit)
```

**핵심 설계 원칙**:
- 모든 경로는 `path_utils.StudyPaths`를 통해 중앙 관리
- LLM 호출은 반드시 `LLMRouter.generate()`를 경유
- 파일 조작(퀴즈 이동/저장)은 반드시 `quiz_store.py`를 경유
- 학습 결과 기록은 반드시 `MemoryManager.record_result()`를 경유

## 3. 10단계 파이프라인 상세

**진입점**: `synthesize.process_note(note_path, config, logger)`

```
process_note()
│
├─ [전처리] detect_subject()  ← 파일 경로에서 과목 키 추출
│         └─ folder_mapping 역참조: "유기화학/" → "organic_chem"
│
├─ [1단계] collect_sources_for_note()
│         └─ SourceAggregator.aggregate_for_note()
│              ├─ note_text: .md 파일 전체 내용
│              ├─ textbook_text: pdfplumber(기본) / pymupdf(fallback) / marker-pdf(고품질)
│              ├─ slides_text: PPTX → python-pptx, PDF → pdfplumber
│              ├─ textbook_images: PDF 이미지 추출
│              └─ slides_images: PPTX/PDF 이미지 추출
│
├─ [2단계] marker-pdf 상태 확인 (실제 변환은 1단계에서 완료)
│         └─ _is_marker_available() → marker 설치 여부 로그
│
├─ [3단계] run_agents()
│         ├─ ClassifierAgent.classify(note_text, subject)
│         │    └─ 출력: {sections: [{title, key_concepts, difficulty}], overall_difficulty}
│         └─ GapDetector.detect_gaps(note_text, subject, textbook_text, slides_text)
│              └─ 출력: {gaps: [{concept, description}], coverage_score}
│
├─ [4단계] run_cross_subject_analysis()
│         └─ CrossSubjectAgent.find_connections(note_text, subject, other_subjects)
│              └─ 출력: {connections: [{source_concept, target_subject, target_concept}]}
│
├─ [5단계] run_paper_enrichment()
│         └─ paper_fetcher.fetch_papers_for_note(note_text, subject, config)
│              ├─ Semantic Scholar API → 인용 수 기준 정렬
│              ├─ PMC 오픈액세스 체크 → PDF 다운로드 (cache/papers/)
│              └─ 출력: [{title, abstract, full_text(선택), citation_count}]
│
├─ [6단계] build_synthesis()
│         ├─ synthesize_notes() — 2-pass 또는 1-pass
│         │    ├─ LM Studio 활성 시 2-pass:
│         │    │    ├─ Pass 1: LM Studio (task="draft") → 초안
│         │    │    └─ Pass 2: Claude (task="synthesis_deep") → 심화
│         │    └─ LM Studio 비활성 시 1-pass:
│         │         └─ Claude (task="synthesis_deep") → 직접 종합
│         └─ add_pubmed_section() — PubMed 검색 결과 섹션 추가
│              └─ pubmed_client.search_and_summarize() → Claude overview
│
├─ [7단계] append_analysis_sections()
│         └─ StudyAnalyst (ChatGPT)
│              ├─ generate_gap_supplements(gaps) → 갭 개념 보충 설명
│              ├─ analyze_cross_subject(connections) → 교차과목 연결 설명
│              └─ analyze_paper_relevance(papers) → 논문 연관성 분석
│         ※ ChatGPT 실패 시 → add_supplementary_explanations() (Claude fallback)
│
├─ [8단계] append_study_plan_section()
│         ├─ MemoryManager.get_weak_concepts(subject) → 취약 개념 목록
│         ├─ MemoryManager.get_due_reviews(subject) → 오늘 복습 대상
│         └─ StudyAnalyst.generate_study_plan(weak, due, stats) → 학습 계획 섹션
│
├─ [9단계] append_quiz_sections()
│         ├─ textbook_quiz.add_textbook_quiz_section() → 교재 문제 추출 + 삽입
│         └─ generate.process_content() → Claude로 퀴즈 생성 → queue/{quiz_id}.json + .md
│
└─ [10단계] persist_outputs()
          ├─ save_synthesis_md() → output/md/{subject}_{note}.md
          │    └─ vault_inject=true → vault/{과목}/정리/{subject}_{note}.md 복사
          ├─ save_synthesis_pdf() → output/pdf/{subject}_{note}.pdf
          │    └─ pdf_builder.build_pdf() (xhtml2pdf 기반, weasyprint는 Windows에서 미지원)
          ├─ record_synthesis_session() → MemoryManager.record_result() (자동 "correct")
          └─ refresh_hermes_schedule() → HermesAgent.refresh_from_event("synthesis_completed")
```

## 4. LLM 3-tier 라우팅

### 클라이언트 계층

```
LLMRouter
├─ LMStudioClient      ← 로컬 26B 모델 (Tailscale 경유 데스크톱)
├─ ExternalRouterClient ← 외부 라우터 (pbl-router-v4, 선택적)
├─ ChatGPTClient       ← OpenAI GPT (Codex CLI 구독 우선, API fallback)
└─ ClaudeClient        ← Anthropic Claude (Claude Code CLI 구독 우선, API fallback)
```

### 작업별 라우팅 테이블 (config.yaml 기준)

| task_type | 1순위 | 2순위 | 3순위 |
|-----------|-------|-------|-------|
| `caption`, `classify`, `collect`, `draft`, `extract_keywords`, `summarize_draft`, `translate_term` | LM Studio | ChatGPT | Claude |
| `cross_subject`, `gap_analysis`, `paper_analysis`, `study_plan`, `supplement` | ChatGPT | Claude | — |
| `mechanism`, `pubmed_overview`, `quiz`, `quiz_generate`, `synthesis_deep`, `synthesis_final`, `user_response` | Claude only | — | — |
| `general` (미분류) | LM Studio | ChatGPT | Claude |

### Claude CLI 모델 오버라이드 (config.yaml)

| task_type | CLI 모델 |
|-----------|----------|
| `mechanism`, `synthesis_deep`, `synthesis_final`, `user_response` | opus |
| `pubmed_overview`, `quiz`, `quiz_generate` | sonnet |
| 기본값 | sonnet |

### thinking/reasoning 자동 활성화

```python
# synthesize.py → llm_router.py:611-645
_THINKING_TASKS = {"synthesis_deep", "synthesis_final", "mechanism", "gap_analysis"}
_HIGH_REASONING_TASKS = {"synthesis_deep", "mechanism", "quiz_generate"}

# 로직:
use_thinking = task_type in _THINKING_TASKS or reasoning_effort == "high"
# → ClaudeClient._call_api(use_thinking=True) 호출 시 extended thinking 활성화
# → thinking 모드에서는 temperature 강제 1.0 (Anthropic API 요구사항)
# → thinking_budget은 min(config.thinking_budget, max_tokens - 1024) 로 클램프
```

### 구독 CLI 호출 방식

**Claude**: `claude -p {prompt} --model {cli_model} --output-format text`
- 환경에서 `ANTHROPIC_API_KEY`를 제거한 env로 실행 (API 우회 방지)
- 타임아웃: 180초

**ChatGPT (Codex)**: `codex exec {prompt}` 또는 `node codex.js exec {prompt}`
- PATH → npm global → Windows 설치 경로 순으로 탐색
- 타임아웃: 120초

### LLMRouter.generate_json()

JSON 응답이 필요한 에이전트용. `generate()` 호출 후 응답에서 ```json 블록 또는 중괄호 구간 추출, `json.loads()` 파싱. 파싱 실패 시 `None` 반환.

## 5. 메모리 시스템과 SM-2 알고리즘

### 저장 이중화

```
MemoryManager
├─ weak_concepts.json (항상 사용)
│    └─ {subject: {concept_tag: ConceptEntry}}
├─ cache/learning_history.json (항상 사용)
│    └─ {events: [...최근 500개...]}
└─ mem0 Chroma 벡터 DB (optional, enabled=true 시)
     └─ 100.78.18.95:8000 (Tailscale 경유 데스크톱 Chroma)
```

### ConceptEntry 스키마

```json
{
  "encounter_count": 정수,
  "correct_count": 정수(부분정답 +0.5),
  "last_encounter": "ISO8601",
  "mastery": 0.0~1.0,
  "priority": "high|medium|low",
  "related_notes": ["노트파일명", ...],
  "recent_mistakes": [{"date", "result", "memo"}, ...최근5개],
  "sr_interval": 정수(일),
  "sr_ease_factor": 실수,
  "sr_next_review": "YYYY-MM-DD"
}
```

### mastery 계산식

```
mastery = correct_count / encounter_count

priority:
  mastery < 0.5  → "high"
  mastery < 0.8  → "medium"
  mastery ≥ 0.8  → "low"
```

### SM-2 변형 알고리즘 (memory_manager.py:258–281)

| 결과 | interval 갱신 | ease_factor(ef) 갱신 |
|------|--------------|----------------------|
| correct (interval=1) | **6일** | ef + 0.1 |
| correct (interval 2~3) | 6일 | ef + 0.1 |
| correct (interval > 3) | int(interval × ef) | ef + 0.1 |
| partial | max(1, int(interval × 0.7)) | ef − 0.1 |
| wrong | 1일 | ef − 0.3 |

- ef 하한: 1.3 (모든 경우에 `max(1.3, ...)` 적용)
- `sr_next_review = 오늘 + interval일`

> **수정 이력**: 최초 구현에서 `correct (interval=1)` 케이스가 `interval = 1`로 유지되어 첫 번째 정답 후에도 익일 복습으로 처리되는 버그 존재 → 2026-04-17 `interval = 6` 으로 수정 (memory_manager.py:265)

### get_due_reviews() 기준

`sr_next_review <= today` 인 개념을 priority(high→low) + mastery(낮은 순)으로 정렬 반환.

### study_explain_concept 설명 깊이 규칙

```
mastery < 0.5  → 기초부터 상세히, 비유·예시 풍부
0.5 ≤ mastery < 0.8 → 핵심 포인트 중심, 빠진 부분 보충
mastery ≥ 0.8 → 심화·응용, 시험 출제 포인트 강조
```

### 매 호출마다 config 재로드

`mcp_server.py`의 모든 Tool/Resource 함수는 `_load_config()`를 매번 호출함. 서버 재시작 없이 config.yaml 변경이 즉시 반영되는 의도적 설계.

## 8. 경로 해석 규칙

`path_utils.get_study_paths(config)` → `StudyPaths` dataclass 반환.

```python
vault       = resolve_path(config["vault_path"], cwd)
notes_base  = resolve_path(config["notes_dir"], vault)  # 상대면 vault 기준
pipeline    = resolve_path(config["pipeline_dir"], vault)
```

`resolve_path(value, base)`: 절대경로면 그대로 사용, 상대경로면 base 기준으로 `.resolve()`.
