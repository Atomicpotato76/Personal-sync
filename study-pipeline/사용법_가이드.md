# Study Pipeline 사용법 가이드

> AI가 공부 자료를 읽고 Markdown 요약 노트, 퀴즈, 복습 일정을 만들어주는 개인 학습 시스템입니다.

---

## 기본 흐름

```
필기/교재 PDF/강의 PDF를 선택해서 실행
        ↓
AI가 자료를 합쳐 Markdown 정리노트 생성
        ↓
퀴즈를 생성해서 queue/에 저장
        ↓
퀴즈 결과를 기록하면 로컬 JSON에 취약 개념과 복습 일정 저장
        ↓
Hermes 또는 대시보드에서 복습 계획 확인
```

---

## 기능 1 — 파이프라인 수동 실행

자동 감시와 Windows 스케줄러는 제거되었습니다. 이제 실행은 CLI 또는 GUI에서 명시적으로 시작합니다.

### 단일 노트

```powershell
python scripts/synthesize.py "<노트.md>"
```

### 폴더 전체

```powershell
python scripts/synthesize.py "<폴더 경로>"
```

### 챕터 통합

```powershell
python scripts/synthesize.py --chapter organic_chem ch4
python scripts/synthesize.py --chapter-all organic_chem
```

### PDF만으로 처리

```powershell
python scripts/synthesize.py --pdf-only "Chapter 1.pdf" --subject mycology
```

### 필기와 PDF 직접 지정

```powershell
python scripts/synthesize.py --notes note1.md note2.md --textbook textbook.pdf --slides slides.pdf --subject organic_chem
```

## 기능 2 — Markdown 출력

PDF 출력은 제거되었습니다. 결과물은 Markdown 중심으로 저장됩니다.

| 위치 | 내용 |
|---|---|
| `output/md/` | 생성된 정리노트 |
| Obsidian `{과목}/정리/` | vault 역주입된 정리노트 |
| `output/reports/` | GUI에서 생성한 Markdown 보고서 |

---

## 기능 3 — 퀴즈 생성과 복습

퀴즈 생성은 유지됩니다.

```powershell
python scripts/review.py list
python scripts/review.py do
python scripts/review.py report
```

- `queue/`: 풀기 전 퀴즈
- `approved/`: 완료된 퀴즈
- `rejected/`: 거절한 퀴즈
- `weak_concepts.json`: 개념별 숙련도와 다음 복습일
- `cache/learning_history.json`: 전체 학습 이력

---

## 기능 4 — 로컬 JSON 학습 기억

원격 Chroma/mem0 저장소는 사용하지 않습니다. 학습 기록은 로컬 JSON으로만 관리합니다.

| 파일 | 내용 |
|---|---|
| `weak_concepts.json` | 개념별 mastery, priority, 복습 간격 |
| `cache/learning_history.json` | 퀴즈/학습 이벤트 기록 |
| `cache/pending_links.json` | 교차과목 연결 검토 대기 |
| `cache/note_index.json` | 노트 인덱스 |

---

## 기능 5 — 논문 보강 토글

논문 보강은 기본 OFF이며, 필요할 때 GUI 설정에서 켤 수 있습니다.

| 설정 | 의미 |
|---|---|
| `papers.enabled` | Semantic Scholar 기반 논문 수집 |
| `pubmed.enabled` | PubMed 기반 논문 overview 추가 |

시험용 빠른 정리에서는 OFF를 권장합니다. 연구 배경까지 붙이고 싶을 때만 ON으로 켜면 됩니다.

---

## 기능 6 — 웹 대시보드

GUI는 유지됩니다.

```powershell
run_dashboard.bat
```

또는:

```powershell
streamlit run scripts/dashboard/app.py
```

대시보드에서 할 수 있는 일:

- 파이프라인 실행
- PDF Only 실행
- 퀴즈 승인/거절
- 설정 편집
- Markdown 보고서 생성
- Hermes 일정 확인

---

## 기능 7 — Hermes 일정

```powershell
python scripts/hermes.py plan-day
python scripts/hermes.py plan-week
python scripts/hermes.py add-exam organic_chem 2026-05-10 "중간고사"
```

Hermes는 로컬 JSON에 저장된 취약 개념과 시험 일정을 기준으로 공부 계획을 만듭니다.

## 제거된 기능

| 제거 항목 | 이유 |
|---|---|
| 외부 router 서버 연동 | 내부 LM Studio/OpenAI/Claude 라우팅으로 충분 |
| Claude Desktop MCP | 더 이상 사용하지 않음 |
| Obsidian watcher | 자동 실행보다 명시적 실행 우선 |
| Windows scheduler | 자동 실행 관리비 제거 |
| PDF 출력 | Markdown 단일 출력으로 단순화 |
| 원격 Chroma/mem0 | 로컬 JSON으로 단순화 |
